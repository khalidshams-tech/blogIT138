# Week 1: Starting My Technical Blog

## Background

For my IT138 Linux+ class, I am creating a technical blog to document what I learn each week. This blog will help me practice writing about Linux, troubleshooting, cybersecurity, cloud computing, and IT support in a professional way.

Technical skills are important, but communication is also important. A good IT professional must be able to explain problems, document solutions, and help other people understand technical steps.

## Objective

The goal for Week 1 was to start building a professional learning portfolio using GitHub Pages. I wanted the website to be simple, organized, and useful for my instructor, classmates, and future employers.

## Problem

At the beginning, the main challenge was understanding how to organize the website clearly. If all posts are placed on the home page, the site can become confusing. A portfolio should guide visitors quickly to the right information.

## Solution

I created a separate **Linux (IT138)** section for weekly blog posts. This keeps the home page focused on who I am and what I am learning, while the Linux page becomes the main place for class blog content.

## What I Learned

This week I learned that a technical portfolio should be organized with purpose. A clean structure makes the website easier to read and shows professionalism.

I also learned that documenting work early is valuable. Even simple weekly notes can become evidence of progress, discipline, and technical growth over time.

## Next Steps

Next, I will continue adding weekly posts that explain what I learn in Linux system administration, troubleshooting, and real-world IT skills.
